# MUSIC ARCHIVE — 프로젝트 기술 문서

## 1. 프로젝트 개요
본 프로젝트는 별도의 서버(백엔드) 없이 HTML, CSS, JavaScript만으로 구동되는 **100% 프론트엔드 정적(Static) 웹 애플리케이션**입니다. 
사용자는 장르, 발매 연도, 아티스트별로 큐레이션된 음악 앨범 목록을 탐색하고, 각 앨범의 트랙리스트, 참여진 코멘터리, 컨셉아트, 시대배경 등을 심층적으로 열람할 수 있습니다.

## 2. 모듈(디렉토리) 구조 및 파일 역할

### 📄 HTML 문서 (View & 라우팅 엔드포인트)
* `index.html`: **메인 홈 화면** - 장르 그리드 렌더링 구역
* `genre.html`: **장르 상세 화면** - 선택된 특정 장르에 속한 앨범 카드 목록 출력
* `year.html`: **연도별 탐색 화면** - 선택된 발매 연도에 속한 앨범 필터링 출력
* `artist.html`: **아티스트 상세 화면** - 사진, 바이오(출신, 활동기간) 정보와 해당 아티스트가 발표한 앨범 리스트 출력
* `album.html`: **앨범 상세 화면** - 포스터 레이아웃 구조, 트랙 및 참여 크레딧 등 다중 탭 인터페이스 렌더링

### 🗃️ JavaScript (Data & Logic)
* `data.js`: **정적 데이터베이스** - `GENRES`, `ALBUMS`, `ARTISTS` 배열을 전역 상수로 관리합니다. 기존 음악 사이트들의 DB를 벤치마킹하여 앨범 커버 경로, 텍스트 정보들이 압축되어 하드코딩된 저장소입니다.
* `app.js`: **메인 컨트롤러(엔진)** - 브라우저의 URL 주소(Query 파라미터)를 읽어 이 화면에 어떤 데이터를 그려야 할지 응답하는 시스템의 코어 로직입니다.
* `user_log.js`: **시스템 로깅 파일** - 작업자(AI)와 사용자 간의 지시사항 및 변경 내역이 append-only(원자적 추가) 형태로 무결성 있게 영구 보존됩니다. 

### 🎨 CSS 및 시스템 에셋
* `styles.css`: 전역 스타일시트. 프리미엄 다크 테마 컬러 파레트, Flexbox/Grid 기반의 반응형 카드 뷰 레이아웃, 호버 트랜지션 효과를 통제합니다.
* `img/` 폴더: 오프라인용으로 저장된 앨범 커버 및 아티스트 사진 원본 에셋이 보관되어 프론트엔드의 화상 자원을 공급합니다.

## 3. 핵심 시스템 아키텍처 (`app.js` 동작 원리)

본 솔루션의 가장 큰 특징은 **Javascript 단일 엔트리 포인트(PAGE Detector)** 기술의 도입입니다.

### 로직 흐름 (Data Flow)
1. **Request**: 사용자가 앨범 카드를 클릭하면 `album.html?id=flower-boy` 형태의 링크로 이동합니다.
2. **Detection**: `album.html` 로드 완료 후 페이지 최하단에서 `app.js` 소스가 구동됩니다. 스크립트 최상단 `const PAGE` 지역 변수가 현재 URL의 `pathname`을 스캔하여 자신이 속한 View 공간(index, artist, album 중 하나)을 식별합니다.
3. **Trigger**: `DOMContentLoaded` 이벤트가 발동하며, 식별된 View 공간 전용의 메인 실행 함수(예: `initAlbum()`)를 호출합니다.
4. **Parsing**: `initAlbum()` 내부에서 `URLSearchParams`를 통해 쿼리 주소 맨 끝의 `?id=flower-boy` 값을 추출합니다.
5. **Fetching & Querying**: 추출한 아이디 값을 검색 키워드로 삼아, `data.js`의 `ALBUMS` 배열에 저장된 수많은 자료를 뒤져 일치하는 하나의 앨범 데이터 객체를 확보합니다.
6. **Rendering**: 확보한 앨범 데이터 객체(포스터 이미지, 연도, 제목 등)를 템플릿 리터럴(Template Literals) 문법으로 가공해 동적인 HTML 마크업 문자열로 조립한 뒤, 메인 화면의 `div#album-poster` 영역에 `innerHTML` 명령어로 덮어씌워 화면 송출을 완결 짓습니다.

## 4. 데이터 스키마 명세서 (`data.js`)

### 1️⃣ `GENRES` 배열 (카테고리 스키마)
* 고유값 `id`, 화면 노출명 `name`, 테마 컬러 `color`, 배경 효과 값인 `bg` 필드를 지닙니다. 구조의 뼈대를 형성합니다.

### 2️⃣ `ALBUMS` 배열 (코어 엔티티 스키마)
* `id` 및 외래키 성격의 `genre`, `year`, `artist` 필드를 통해 카테고리 간 참조 관계를 맺고 있습니다.
* 포스터에 쓰이는 `coverImage`, 탭 UI에서 뻗어나가는 `concept`, `era`, `credits`, `tracks` 객체 배열 등 무거운 정보를 보관하는 초대형 데이터 저장소 구조를 띱니다.

### 3️⃣ `ARTISTS` 배열 (메타 엔티티 스키마)
* 특정 가수 전용 상세 페이지를 구축할 수 있도록 `photo`와 인물 상세 이력인 `bio`, 데뷔일인 `activeFrom`, 활동 무형인 `origin` 필드로 구성된 백과사전 딕셔너리입니다.
